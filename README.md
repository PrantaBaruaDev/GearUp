# **🏋️GearUp**

GearUp is a backend API for a gear rental and rental management platform. It allows customers to browse gear items, providers to manage their inventory, admins to oversee the system, and users to make rentals and payments.

## Project Overview

This project is built as a RESTful API using Node.js, Express.js, TypeScript, Prisma ORM, and PostgreSQL. The system supports:

*   User authentication and role-based authorization
*   Customer, Provider, and Admin accounts
*   Gear item browsing and management
*   Rental order creation and tracking
*   Payment flow with Stripe integration
*   Review creation and moderation
*   Category management

## Tech Stack

### Core

*   Node.js

*   TypeScript
*   Prisma ORM
*   PostgreSQL

## Model Design Plan

The database is designed around a rental marketplace flow.

### Main Models

Users

*   Stores user identity and role
*   Roles: CUSTOMER, PROVIDER, ADMIN

| **Field** | **Requirement (Plain Text)** |
| --- | --- |
| `id` | String (UUID), Primary Key, Autogenerated. |
| `name` | String, Required. |
| `email` | String, Required, Must be unique. |
| `password` | String, Required. |
| `role` | Enum (`Role`), Defaults to `CUSTOMER`. |
| `status` | Enum (`ActiveStatus`), Defaults to `ACTIVE`. |
| `created_at` | DateTime, Defaults to current timestamp (Timestamp 6 precision). |
| `updated_at` | DateTime, Automatically updates on modification (Timestamp 6 precision). |

Profiles

*   Stores profile details such as photo, address, and phone
*   Linked one-to-one with Users

| **Field** | **Requirement (Plain Text)** |
| --- | --- |
| `id` | String (UUID), Primary Key, Autogenerated. |
| `userId` | String, Required, Must be unique (Foreign Key to `Users.id`). |
| `profilePhoto` | String, Optional / Nullable. |
| `address` | String, Required. |
| `phone` | String, Required, Maximum length 20 characters. |
| `created_at` | DateTime, Defaults to current timestamp (Timestamp 6 precision). |
| `updated_at` | DateTime, Automatically updates on modification (Timestamp 6 precision). |

Category

*   Groups gear items into categories

| **Field** | **Requirement (Plain Text)** |
| --- | --- |
| `id` | String (UUID), Primary Key, Autogenerated. |
| `name` | String, Required, Must be unique. |
| `createdAt` | DateTime, Defaults to current timestamp. |

GearItems

*   Represents available rental gear
*   Belongs to one Category and one Provider (Users)

| **Field** | **Requirement (Plain Text)** |
| --- | --- |
| `id` | String (UUID), Primary Key, Autogenerated. |
| `title` | String, Required. |
| `description` | String, Required. |
| `brand` | String, Required. |
| `pricePerDay` | Decimal value, Required. |
| `stock` | Integer numerical value, Defaults to `1`. |
| `availableStock` | Integer numerical value, Defaults to `1`. |
| `categoryId` | String, Required (Foreign Key linking to `Category.id`). |
| `providerId` | String, Required (Foreign Key linking to `Users.id`). |
| `createdAt` | DateTime, Defaults to current timestamp (Timestamp 6 precision). |
| `updatedAt` | DateTime, Automatically updates on modification (Timestamp 6 precision). |

RentalOrders

*   Stores a rental booking request from a customer
*   Contains startDate, endDate, totalPrice, and status

| **Field** | **Requirement (Plain Text)** |
| --- | --- |
| `id` | String (UUID), Primary Key, Autogenerated. |
| `customerId` | String, Required (Foreign Key linking to `Users.id`). |
| `startDate` | DateTime, Explicitly limited to precise Date storage formats. |
| `endDate` | DateTime, Explicitly limited to precise Date storage formats. |
| `totalPrice` | Decimal calculation value, Required. |
| `status` | Enum (`OrderStatus`), Defaults to `PENDING`. |
| `createdAt` | DateTime, Defaults to current timestamp (Timestamp 6 precision). |
| `updatedAt` | DateTime, Automatically updates on modification (Timestamp 6 precision). |

RentalItems

*   Junction details for products included in a rental order

| **Field** | **Requirement (Plain Text)** |
| --- | --- |
| `id` | String (UUID), Primary Key, Autogenerated. |
| `rentalOrderId` | String, Required (Foreign Key linking to `RentalOrders.id`). |
| `gearItemId` | String, Required (Foreign Key linking to `GearItems.id`). |
| `quantity` | Integer numerical value, Defaults to `1`. |

Payments

*   Stores payment status and transaction records for each rental order

| **Field** | **Requirement (Plain Text)** |
| --- | --- |
| `id` | String (UUID), Primary Key, Autogenerated. |
| `userId` | String, Required, Must be unique (Foreign Key tracking to `Users.id`). |
| `rentalOrderId` | String, Required, Must be unique (Foreign Key tracking to `RentalOrders.id`). |
| `amount` | Decimal financial calculations balance format, Required. |
| `paidAt` | DateTime, Optional / Nullable processing timestamp tracking marker. |
| `status` | Enum (`PaymentStatus`), Defaults to balance structure tracking parameter `PENDING`. |
| `stripeCustomerId` | String tracker token string expression, Optional / Nullable. |
| `stripeTransactionId` | String tracker token string expression, Optional / Nullable. |
| `createdAt` | DateTime, Defaults to current execution timestamp environment (Timestamp 6 precision). |
| `updatedAt` | DateTime, Automatically updates on modifications framework (Timestamp 6 precision). |

Reviews

*   Stores customer feedback and ratings for gear items

| **Field** | **Requirement (Plain Text)** |
| --- | --- |
| `id` | String (UUID), Primary Key, Autogenerated. |
| `rating` | Integer numerical value, DB Check constraint enforcing values `1` to `5`. |
| `comment` | String, Optional / Nullable. |
| `customerId` | String, Required (Foreign Key linking to `Users.id`). |
| `gearItemId` | String, Required (Foreign Key linking to `GearItems.id`). |
| `createdAt` | DateTime, Defaults to current execution timestamp. |

*   RentalOrders 1-to-1 Payments
*   GearItems 1-to-many Reviews

### Database Constraint Example

Rating must be between 1 and 5:

```
ALTER TABLE "reviews"
ADD CONSTRAINT "reviews_rating_range_check"
CHECK (rating >= 1 AND rating <= 5);
```

## Authentication & Authorization

The API uses JWT-based authentication. After login, the server sets access and refresh tokens in HTTP-only cookies.

Supported roles:

*   CUSTOMER
*   PROVIDER
*   ADMIN

### Auth Endpoints

*   POST /api/auth/register
*   POST /api/auth/login
*   GET /api/auth/me
*   POST /api/auth/logout
*   POST /api/auth/refresh-token

## API Endpoints

Below are the main endpoints with example request bodies and response JSON formats.

### Register User

POST /api/auth/register

Request body:

```
{
  "name": "Pranta Barua",
  "email": "pranta.admin@mail.com",
  "password": "1234",
  "role": "ADMIN",
  "profilePhoto": "https://example.com/profile.jpg",
  "address": "Chittagong",
  "phone": "012564565"
}
```

Example response:

```
{
  "success": true,
  "statusCode": 201,
  "message": "User registered successfully",
  "data": {
    "id": "uuid",
    "name": "Pranta Barua",
    "email": "pranta.admin@mail.com",
    "role": "ADMIN"
  }
}
```

### Login User

POST /api/auth/login

Request body:
